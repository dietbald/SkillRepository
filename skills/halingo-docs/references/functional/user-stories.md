# User Stories: Halingo SaaS

This document outlines the initial set of user stories for the Halingo practice management platform.

## 1. Practice Management (Dashboard)
| ID | User Story | Acceptance Criteria |
| :--- | :--- | :--- |
| **US.01** | **As a** Speech Therapist, **I want** a centralized dashboard, **so that** I can get an immediate overview of my practice status (appointments, pending tasks). | - Dashboard displays key metrics (e.g., today's sessions).<br>- Visual indicators for urgent administrative tasks. |
| **US.02** | **As a** Practice Owner, **I want** to see an overview of multiple therapists' schedules (if applicable), **so that** I can manage the clinic's resources efficiently. | - Ability to toggle views between different therapists.<br>- Conflict detection in scheduling. |

## 2. Patient Administration
| ID | User Story | Acceptance Criteria |
| :--- | :--- | :--- |
| **US.03** | **As a** Speech Therapist, **I want** to create and manage digital patient dossiers, **so that** I have all clinical and administrative data securely stored in one place. | - Secure storage of personal data (GDPR).<br>- Ability to add/edit patient information (contact, medical history). |
| **US.04** | **As a** Therapist, **I want** to access patient files from any device, **so that** I can review history during school or home visits. | - Responsive UI for mobile/tablet.<br>- Data synchronization across devices. |

## 3. Scheduling & Agenda
| ID | User Story | Acceptance Criteria |
| :--- | :--- | :--- |
| **US.05** | **As a** Therapist, **I want** an intuitive calendar to book and move sessions, **so that** managing my daily schedule is effortless. | - Drag-and-drop session scheduling.<br>- Different views (day, week, month). |
| **US.06** | **As a** Therapist, **I want** automated session reminders for patients (inferred), **so that** I can reduce the number of "no-shows". | - Email/SMS notification settings for appointments. |

## 4. Financial & Invoicing
| ID | User Story | Acceptance Criteria |
| :--- | :--- | :--- |
| **US.07** | **As a** Therapist, **I want** to generate invoices automatically based on completed sessions, **so that** I spend less time on manual billing. | - One-click invoice generation for sessions.<br>- Support for "De Conventie" tariff structures. |
| **US.08** | **As a** Therapist, **I want** to track the payment status of my invoices, **so that** I can easily identify outstanding payments. | - Status indicators (Paid, Pending, Overdue).<br>- Payment recording functionality. |
