# Todos

## What this area covers

The lightweight per-user task list a therapist keeps inside Halingo for personal reminders — notes to self that don't belong on a patient dossier or a calendar slot.

## Features in this area

- Create / read / update / delete todos — add a task, mark it done, edit, delete.

## Key product behaviors

- Each user has their own todo list. Items are scoped strictly to the user — a colleague (even the practice owner) cannot see another therapist's todos.
- Todos are not patient-bound and not appointment-bound; they are free-text reminders.
- The dashboard renders the user's open todos in a tile (see Main Dashboard) for quick triage on login.
- There is no due-date / priority / category structure — the list is intentionally simple.

## Cross-references

- Main Dashboard — the open-todo count and the most-recent items are surfaced in a dashboard tile.
- Identity — items are scoped per user; the user's identity determines what they see.
