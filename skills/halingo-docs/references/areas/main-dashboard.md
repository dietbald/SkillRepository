# Main Dashboard

## What this area covers

The post-login landing screen for every user — a widget grid that gives an at-a-glance overview of the day, the week, and any items needing attention.

## Features in this area

- Layout — the grid arrangement of the dashboard widgets and the responsive behaviour on smaller screens.
- Statistics band — a row of summary KPIs at the top (today's sessions, weekly bar chart, busiest / quietest / average day).
- Notifications tile — a count of unread in-app notifications with a link to the inbox.
- Open invoices tile — count and total of unpaid patient invoices, with a link to the financial overview.
- Todos tile — the user's personal to-do list (see Todos area).
- Newsfeed tile — the latest system-wide announcements published by Halingo.

## Key product behaviors

- The dashboard is the same for every role; it is not currently customisable per user, and its layout does not vary by role.
- The "today's sessions" card lists the current day's appointments in chronological order, with a quick link into the calendar for context.
- The weekly bar chart shows the user's session counts across a working week (typically Mon–Sat) so the therapist can spot under-utilised slots.
- The busiest / quietest / average values are derived from the same week as the bar chart.
- Counts on the notifications, todos, and invoice tiles update as the underlying data changes during the session.

## Cross-references

- In-app Notifications — the source of the count and the destination of the "see all" link.
- Todos — the per-user to-do list rendered in the dashboard tile.
- Smart Invoicing — the open-invoices tile reflects the same data shown on the financial overview screen.
- Newsfeed — the announcement stream embedded in the dashboard tile.
- Practice Analytics — deeper drill-downs into the same KPIs live in the analytics screens.
